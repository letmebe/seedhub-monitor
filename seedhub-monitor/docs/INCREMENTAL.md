# 增量更新功能说明

## 更新逻辑

系统采用智能增量更新策略，避免重复处理：

### 抓取阶段（scrape.js）

```
新抓取电影数据 → 检查数据库
                    ↓
         ┌──────────┴──────────┐
         │                     │
    影片不存在?            影片已存在?
         │                     │
    INSERT新记录         检查转存状态
         │                     │
    ✅ 新增                ├─ 已转存 → ⏭️ 跳过
                          │
                          └─ 未转存 → 检查链接
                                        │
                                   ├─ 链接相同 → ⏭️ 跳过
                                   │
                                   └─ 链接不同 → 🔄 更新链接
```

### 场景示例

**场景1：新电影首次抓取**
- 数据库无记录 → INSERT → 状态: 未转存

**场景2：已转存电影重复抓取**
- 数据库有记录且transferred=1 → 跳过，不做任何更新

**场景3：未转存电影，链接未变**
- 数据库有记录，transferred=0，链接相同 → 跳过

**场景4：未转存电影，链接已更新**（典型场景）
- 数据库有记录，transferred=0，链接不同 → UPDATE更新链接
- 原因：早期资源清晰度不够，用户未转存；后期更新了更好的资源链接

```bash

# 或
node scrape.js --loop  # 每24小时运行一次
```

## 数据库字段说明

| 字段 | 说明 | 用途 |
|-----|------|-----|
| `title` | 电影标题 | 唯一标识（与pan_type组合） |
| `pan_type` | 网盘类型 | 百度/UC/夸克/天翼 |
| `pan_link` | 网盘链接 | **用于判断是否更新** |
| `transferred` | 是否已转存 | 0=未转存, 1=已转存 |
| `updated_at` | 更新时间 | 链接更新时更新此字段 |

---

## 关键优势

### 1. 智能跳过
- 已转存电影不会被覆盖
- 链接未变不重复处理

### 2. 资源升级感知
- 自动检测链接更新
- 显示更新详情
- 用户可选择转存新资源

### 3. 用户主导
- **交互式选择**：用户决定转存哪些
- 支持分批转存
- 支持指定ID

### 4. 数据安全
- JSON备份保留
- 数据库持久化
- Git版本控制

## 典型工作流

```
第1天:
  抓取 → 发现"飞驰人生3"(TS版) → 新增 → 用户查看后觉得清晰度不够 → 不转存

第3天:
  抓取 → 发现"飞驰人生3"(链接变了) → 更新链接 → 用户查看 → 转存

第7天:
  抓取 → 发现"飞驰人生3"(链接相同) → 跳过 → 数据不变

长期运行:
  数据库持续积累 → 用户定期筛选转存 → 增量更新避免重复
```

## API接口

### upsertMovie(movie)

单部电影增量更新：

```javascript
const db = require('./db');

await db.initDatabase();

const result = db.upsertMovie({
  title: '飞驰人生3',
  url: 'https://www.seedhub.cc/movies/134742/',
  panType: '百度',
  panLink: 'https://pan.baidu.com/s/1xxx',
  extractCode: 'ojmk'
});

console.log(result);
// { action: 'inserted', reason: '新电影' }
// { action: 'skipped', reason: '已转存' }
// { action: 'skipped', reason: '链接相同' }
// { action: 'updated', reason: '链接更新', oldLink: 'https://...' }
```

### upsertMovies(movies)

批量增量更新：

```javascript
const results = db.upsertMovies(movies);

console.log(results);
// {
//   inserted: 5,
//   updated: 2,
//   skipped: 13,
//   details: [...]
// }
```

## 注意事项

1. **唯一约束**：`(title, pan_type)` 组合唯一
2. **链接比较**：精确匹配，不支持正则或模糊匹配
3. **转存保护**：已转存记录永远不会被更新
4. **建议策略**：定期抓取 + 定期手动筛选转存
