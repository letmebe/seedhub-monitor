# Edge调试模式使用指南

## 推荐使用方式

### ✅ 正确做法

**1. 使用独立用户数据目录**
- 已配置：`edge-debug-profile/`
- 与正常Edge完全隔离

**2. 不登录微软账户**
- 调试模式保持独立环境
- 避免同步污染主账户配置

**3. 不开启同步功能**
- 扩展、书签、设置不同步
- 保持调试环境干净

**4. 仅登录必要账户**
- 只登录百度网盘（用于转存）
- 不登录其他个人账户

---

## 工作原理

### 两个独立的Edge实例

```
正常Edge（日常使用）
├─ 用户数据: C:\Users\[用户]\AppData\Local\Microsoft\Edge\User Data
├─ 微软账户: 已登录
├─ 同步: 已开启
├─ 扩展: 个人扩展插件
└─ 用途: 日常浏览、工作等

调试Edge（自动化工具）
├─ 用户数据: [项目目录]\edge-debug-profile
├─ 微软账户: 未登录（推荐）
├─ 同步: 未开启（推荐）
├─ 扩展: 无（干净环境）
└─ 用途: SeedHub抓取、百度网盘转存
```

**两者完全独立，互不影响！**

## 启动步骤

### 1. 首次启动

```bash
cd seedhub-monitor
node start-edge.js
```

会自动创建 `edge-debug-profile/` 目录。

### 2. 首次配置（重要）

启动后，在调试模式Edge窗口中：

**✅ 应该做的**：
- 登录百度网盘（用于自动转存）
- 保持其他设置默认

**❌ 不应该做的**：
- 登录微软账户
- 开启同步功能
- 安装扩展插件
- 修改个人设置

### 3. 日常使用

```bash
# 启动调试模式
node start-edge.js

# 运行抓取
npm start

# 选择性转存
npm run transfer
```

## 配置文件位置

### 调试模式配置

```
seedhub-monitor/
└─ edge-debug-profile/          # 调试模式用户数据
   ├─ Default/
   │  ├─ Cookies               # 百度网盘登录状态
   │  ├─ Preferences           # 设置
   │  └─ Extensions/           # 扩展（应保持空）
   └─ ...
```

### 正常Edge配置（不受影响）

```
C:\Users\[用户名]\AppData\Local\Microsoft\Edge\
└─ User Data/
   └─ Default/
      ├─ Extensions/           # 你的扩展插件
      ├─ Bookmarks             # 书签
      └─ ...                   # 其他个人数据
```

## 注意事项

### ⚠️ 重要提示

1. **首次使用需要登录百度网盘**
   - 在调试模式窗口中登录
   - 登录状态会保存在独立目录
   - 不影响正常Edge的登录状态

2. **不要在调试模式中**
   - 登录微软账户（会同步到云端）
   - 开启同步功能（会污染配置）
   - 安装扩展插件（干扰自动化）
   - 保存个人书签、密码

3. **可以同时使用**
   - 正常Edge和调试Edge可以同时打开
   - 完全独立，互不影响
   - 各自的配置、历史、扩展隔离

## 常见问题

### Q: 为什么不能登录微软账户？

A: 调试模式使用独立配置目录，但如果登录微软账户并开启同步：
- 扩展插件会同步到云端
- 可能污染你的个人账户配置
- 在其他设备上也会同步这些"调试用"数据

### Q: 为什么不需要扩展插件？

A: 自动化工具需要干净环境：
- 扩展可能干扰页面自动化
- 某些扩展会阻止脚本执行
- 保持环境纯净，便于调试

### Q: 可以关闭调试模式窗口吗？

A: 可以，但建议：
- 抓取/转存时保持窗口打开
- 完成后可以关闭
- 下次使用时重新启动：`node start-edge.js`

### Q: 登录状态会丢失吗？

A: 不会，登录状态保存在：
- `edge-debug-profile/Default/Cookies`
- 只要不清除Cookie，登录状态会保留

## 清理建议

### 定期清理（可选）

如果调试模式运行时间较长，可以清理：

```bash
# 清理缓存（保留登录状态）
rm -rf edge-debug-profile/Default/Cache
rm -rf edge-debug-profile/Default/Code Cache

# 完全重置（会丢失登录状态）
rm -rf edge-debug-profile
node start-edge.js  # 重新创建
```

## 推荐工作流

```
第1次使用:
  1. node start-edge.js
  2. 在调试窗口登录百度网盘
  3. 不要登录微软账户，不要开启同步

日常使用:
  1. node start-edge.js       # 启动调试模式
  2. npm start                # 抓取电影
  3. npm run transfer         # 选择转存
  4. 关闭调试窗口             # 完成后可关闭

长期运行:
  - 保持调试窗口打开
  - 定时抓取：npm start（已配置单次运行）
  - 定期手动转存：npm run transfer
```
